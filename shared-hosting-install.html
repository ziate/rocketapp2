دليل تثبيت وإعداد المشروع بالكامل على الاستضافة المشتركة

هذا الدليل يشرح خطوة بخطوة كيفية تجهيز Laravel على استضافة مشتركة بدون Node.js.

1) المتطلبات الأساسية

  • PHP 8.1 أو أحدث.
  • Composer (يفضل على جهازك المحلي).
  • قاعدة بيانات MySQL أو MariaDB.
  • حساب استضافة مشتركة مع cPanel أو مشابه.

2) تجهيز المشروع محليًا

  1. أنشئ مشروع Laravel محليًا:
composer create-project laravel/laravel delivery-app
  1. انسخ ملفات هذا المشروع داخل مجلد Laravel الجديد (استبدال الملفات).
  2. أنشئ ملف البيئة:
cp .env.example .env
php artisan key:generate
يمكنك تعديل إعدادات قاعدة البيانات في ملف .env بعد إنشائها على الاستضافة.

3) إنشاء قاعدة البيانات على الاستضافة

  1. ادخل إلى cPanel > MySQL Databases.
  2. أنشئ قاعدة بيانات جديدة.
  3. أنشئ مستخدمًا جديدًا.
  4. اربط المستخدم بقاعدة البيانات مع صلاحيات كاملة.
  5. احتفظ باسم القاعدة واسم المستخدم وكلمة المرور لاستخدامها في ملف .env.

4) رفع الملفات للاستضافة

  1. ارفع مشروع Laravel بالكامل إلى مجلد خارج public_html (مثال: app).
  2. انقل مجلد public إلى public_html أو حدّد Document Root على public.
  3. عدل مسارات index.php داخل public_html لتشير إلى مجلد المشروع:
require __DIR__.'/../app/vendor/autoload.php';
$app = require_once __DIR__.'/../app/bootstrap/app.php';

5) ضبط ملف البيئة (.env)

  1. افتح ملف .env داخل مجلد المشروع.
  2. حدّث بيانات قاعدة البيانات:
DB_DATABASE=database_name
DB_USERNAME=database_user
DB_PASSWORD=database_password
تأكد من ضبط APP_ENV=production و APP_DEBUG=false.

6) تشغيل المايجريشن

إذا كانت الاستضافة توفر SSH:

php artisan migrate --force

إذا لم يتوفر SSH، استخدم أداة phpMyAdmin لاستيراد قاعدة البيانات بعد تشغيل المايجريشن محليًا.

7) ضبط الصلاحيات

  1. تأكد من صلاحيات الكتابة على:
storage/
bootstrap/cache/

اضبطها على 755 أو 775 حسب الاستضافة.

8) اختبار النظام

  1. افتح رابط الموقع.
  2. تأكد من ظهور لوحة التحكم الرئيسية.
  3. أضف محافظة ومنطقة وعميل ثم جرّب إنشاء طلب.
تم إعداد هذا الدليل لتسهيل نشر المشروع على الاستضافة المشتركة.
